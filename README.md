# FPGA-based Resolver-to-Digital Converter (RDC) System

## 📌 Описание проекта
Проект реализует систему цифровой обработки сигналов для преобразователя «угол-код» (датчик СКВТ). 
**Ключевая особенность:** глубокая математическая оптимизация, позволившая полностью отказаться от аппаратных блоков умножения (DSP slices) в пользу кастомной RTL-логики.

## 📊 Результаты синтеза (Xilinx Spartan-3)
- **Максимальная частота (Fmax):** 84.72 MHz.
- **Запас по частоте:** 69% (для целевой частоты 50 MHz).
- **Использование ресурсов:** ~6% Slices, **0% DSP блоков**.

## 🏗 Архитектура и иерархия
Полная функциональная схема системы доступна в папке [doc/](doc/).

1. **Top Level (`RDC_top.vhd`)** — объединяет все компоненты системы.
2. **Вычислительные модули:**
   - `ADC_UPRAV_DEMOD.vhd` — захват данных с АЦП и демодуляция.
   - `CORDIC.vhd` — итерационный вычислитель угла (использует `ROM_ARCTAN.vhd`).
   - `DDS.vhd` — синтезатор частоты (использует `WIDTH_LUT.vhd`).
3. **Сервисные модули:**
   - `CDC.vhd` — синхронизация сигналов между тактовыми доменами (2-FF Synchronizer).
   - `lcd_contr.vhd` — контроллер отображения данных.
   - `pitanie_SP3_1000.ucf` — файл ограничений (Constraints).

## 🧪 Верификация
В папке `tb/` представлен пример самопроверяющегося тестбенча (**Self-checking Testbench**) для модуля CORDIC. 
Тестбенч автоматически проверяет точность вычислений с помощью конструкций `assert`.

## 🛠 Стек технологий
- **Язык:** VHDL (IEEE 1076-2008, numeric_std).
- **Инструменты:** Xilinx ISE, Vivado Simulator.
